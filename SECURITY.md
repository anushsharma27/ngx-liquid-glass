# Security Policy

## Supported versions

Security fixes are provided for the latest published version of `ngx-liquid-glass`.

| Version | Supported |
| --- | --- |
| Latest release | Yes |
| Older releases | No |

## Reporting a vulnerability

Please do not disclose suspected vulnerabilities in a public issue, discussion, or pull request.

Report them privately through [GitHub's private vulnerability reporting form](https://github.com/anushsharma27/ngx-liquid-glass/security/advisories/new). Include:

- A description of the vulnerability and its potential impact
- A minimal reproduction or proof of concept
- Affected package and Angular versions
- Relevant browser, SSR, or build environment details
- Any suggested remediation, if available

You should receive an acknowledgment within seven days. After validation, the maintainer will coordinate a fix and disclosure timeline. Please allow a reasonable remediation period before publishing details.

## Scope

Reports involving package code, installation schematics, generated package contents, browser DOM handling, SSR behavior, or the demo deployment are in scope. Vulnerabilities in Angular, npm, browsers, or other upstream projects should be reported to the relevant upstream maintainers unless this package introduces the exposure.
